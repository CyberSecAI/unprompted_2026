# Overview

## Summary
Nicolas Lidzborski, Principal Software Engineer at Google, outlines the structural security challenges of GenAI and presents a blueprint for securing agentic workspace environments.

## Key Takeaway
Traditional security boundaries fail because GenAI treats every input token as a potential semantic instruction rather than passive data.

# Insights

## The Semantic Shift
- Traditional AppSec relies on deterministic parsing and syntactic filtering to block known patterns like script tags or SQL keywords.
- In GenAI, every token in the input stream is a potential instruction, making traditional "file and filter" approaches ineffective.
- Attackers utilize linguistic techniques like persuasion and role-playing to bypass filters that expect rigid, well-defined commands.

## Collapsed Control Plane
- Unlike classical computing, LLMs do not structurally separate code (instructions) from data (untrusted input).
- There is no equivalent to an "NX bit" for model memory; any token can potentially overwrite the model's primary instructions.
- This architectural vulnerability requires a shift from monitoring for specific syntax to monitoring for malicious semantic intent.

## Indirect Prompt Injection
- Indirect injection poses a massive risk in productivity tools where instructions are hidden in external content like emails or invites.
- Benign tasks such as "summarize my day" can inadvertently trigger the execution of hidden, malicious payloads in untrusted documents.
- This vector enables scaling attacks, as a single document can silently spread and activate across many users without direct interaction.

## Insecure Output Handling
- Models can be tricked into generating malicious markdown that facilitates silent data exfiltration upon rendering in the user's interface.
- Sensitive data can be leaked through query parameters in image links or other resource-fetching tags generated by the model.
- The "verification gap" means users may only perceive a minor rendering error while their data has already been successfully captured.

## Rogue Actions and the Agency Gap
- The "agency gap" occurs when nondeterministic model reasoning causes an autonomous agent to diverge from the user's actual intent.
- Minor prompt variations can lead to accidental rogue actions, such as sending sensitive data to the wrong recipient due to name similarity.
- Compromised orchestration layers allow attackers to hijack the model's capabilities to perform unauthorized actions on the user's behalf.

## Architecting the Fortress
- Securing GenAI requires a fundamental structural security blueprint rather than relying on reactive, failure-prone semantic filtering.
- Organizations must implement defense-in-depth strategies that specifically account for the inherent "fuzziness" of natural language instructions.
- Clear architectural boundaries must be established to separate trusted system instructions from untrusted external data streams.

## Semantic Resilience
- Building resilience requires rigorous, continuous testing and maintaining human-in-the-loop confirmation for any sensitive or high-impact actions.
- Security teams must develop new methodologies to evaluate and monitor the semantic safety of AI-driven workflows in real-time.
- Proactive structural design is the only way to move beyond the futile "cat and mouse game" of reactive prompt filtering.

## Google Speed and Scale
- Protecting workspace GenAI requires security processes that can match the rapid pace of modern, AI-accelerated software development.
- Three years of GenAI research at Google demonstrates that reactive filtering is insufficient for protecting enterprise-scale sensitive data.
- A structural blueprint is essential for providing durable security guarantees in pervasive, agentic AI environments.
